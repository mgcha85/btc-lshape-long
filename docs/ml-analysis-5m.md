## 분석 목표

5분봉 데이터에서 SL(손절) vs TP(익절) 거래를 사전에 예측할 수 있는가?

### 데이터셋

| 항목 | 값 |
|:-----|:---|
| 전략 설정 | MA48, TP2%, SL1.5% |
| 총 샘플 수 | 620 거래 |
| SL (Label=1) | 340 (54.8%) |
| TP (Label=0) | 280 (45.2%) |
| 피처 수 | 177개 |

{: .note }
1시간봉(165개)보다 **3.8배 많은 샘플**을 확보했으나, 클래스 분포가 더 균형적입니다.

## 왜 5분봉 ML이 작동하지 않는가?

### 1. 노이즈 대비 신호 비율

```
1시간봉: 신호 > 노이즈 → 패턴 학습 가능
5분봉:   노이즈 >> 신호 → 패턴 학습 어려움
```

### 2. 시장 미세구조

- 5분봉에서는 **랜덤 변동**이 SL/TP 결과를 좌우
- 기술적 지표의 예측력이 단기에서 급격히 감소
- HFT/알고리즘 거래의 노이즈가 지배적

### 3. 피처 유효성

| 피처 유형 | 1시간봉 | 5분봉 |
|:----------|:------:|:-----:|
| 장기 MA 이격도 | 유효 | 약함 |
| BB 폭 | 유효 | 약함 |
| 캔들 패턴 | 보통 | 보통 |
| RSI/MACD | 유효 | 약함 |

## 결론 및 권장사항

### ML 기반 필터 적용

{: .important }
> **ML 필터는 1시간봉에서만 사용하세요.**
>
> | 타임프레임 | ML 필터 적용 | 권장도 |
> |:----------|:------------|:-----:|
> | 1시간봉 | O (AUC 0.62) | **권장** |
> | 5분봉 | X (AUC 0.55) | 비권장 |

### 5분봉 전략 운용

5분봉에서 ML 없이 운용 시:

1. **단순 규칙 기반** 진입/청산만 사용
2. **리스크 관리**에 집중 (적은 TP/SL)
3. **포지션 크기** 축소

### 하이브리드 접근

```python
# 1시간봉에서 ML 필터로 진입 여부 결정
if hourly_ml_filter.predict(hourly_features) == "TP_LIKELY":
    # 5분봉에서 정밀 진입 타이밍 결정
    wait_for_5m_entry_signal()
```
